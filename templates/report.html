Metodología

El proceso de pentesting se llevó a cabo siguiendo una metodología integral que abarca diversas fases para asegurar una cobertura exhaustiva del dominio objetivo. A continuación, se detallan las principales etapas y las herramientas utilizadas:

1. Reconocimiento y Recopilación de Información

  • Subdominios: Se utilizó Amass y otras técnicas de OSINT para descubrir subdominios asociados al dominio principal, ampliando la superficie de ataque a evaluar.
  • Hosts Activos y Huella Digital: Se empleó httpx para identificar hosts activos, determinar sus puertos HTTP/S, y recopilar información sobre tecnologías utilizadas (servidores web, frameworks, etc.).

2. Escaneo de Vulnerabilidades

  • Vulnerabilidades Conocidas (Nuclei): Se ejecutó Nuclei con una amplia gama de plantillas para detectar vulnerabilidades comunes, configuraciones erróneas, y exposiciones de información en los activos identificados.
  • Credenciales Filtradas (HIBP): Se consultó la base de datos de Have I Been Pwned (HIBP) para verificar si alguna dirección de correo electrónico asociada al dominio ha sido comprometida en brechas de seguridad conocidas.
  • Typosquatting: Se utilizó dnstwist para identificar dominios similares al objetivo que podrían ser usados para ataques de phishing o suplantación de identidad.
  • CVEs: Se buscaron Common Vulnerabilities and Exposures (CVEs) relevantes para las tecnologías detectadas, evaluando la presencia de vulnerabilidades conocidas y sus posibles impactos.
  • Escaneo de Puertos (Nmap): Se realizó un escaneo de puertos con Nmap para identificar puertos abiertos, servicios en ejecución y sus versiones, proporcionando una visión de la exposición de la red.
  • Configuración de Seguridad: Se analizaron cabeceras HTTP y otras configuraciones del servidor web para identificar posibles debilidades como la exposición de información sensible o la falta de políticas de seguridad.

3. Análisis y Reporte

  • Correlación de Hallazgos: Los resultados de todas las herramientas se correlacionaron para obtener una visión unificada de la postura de seguridad.
  • Evaluación de Riesgos: Cada hallazgo se evaluó en términos de severidad y probabilidad de explotación para determinar el riesgo global.
  • Recomendaciones: Se generaron recomendaciones claras y accionables para mitigar las vulnerabilidades identificadas, priorizando aquellas con mayor impacto.

Aunque se esfuerza por identificar la mayor cantidad posible de vulnerabilidades, no garantiza una cobertura del 100% de todas las posibles debilidades de seguridad. Glosario de Términos

Pentesting (Prueba de Penetración):
Simulación de un ataque cibernético contra un sistema informático para encontrar vulnerabilidades de seguridad.
Vulnerabilidad:
Una debilidad en un sistema que puede ser explotada por un atacante para comprometer la seguridad.
Exploit:
Un fragmento de software, datos o secuencia de comandos que aprovecha una vulnerabilidad para causar un comportamiento no deseado o imprevisto en el software o hardware de un ordenador.
Severidad (Critical, High, Medium, Low, Informational):
Clasificación del impacto potencial de una vulnerabilidad. Crítica es la más grave, Informacional la menos.
OSINT (Open Source Intelligence):
Recopilación de información de fuentes públicas y abiertas.
Subdominio:
Una división de un dominio principal (ej. blog.ejemplo.com es un subdominio de ejemplo.com).
Nmap:
Herramienta de código abierto utilizada para el descubrimiento de redes y auditorías de seguridad.
Nuclei:
Escáner de vulnerabilidades basado en plantillas, utilizado para enviar solicitudes a través de la red para detectar problemas de seguridad.
HTTPX:
Herramienta para probar múltiples servidores HTTP/S, identificar tecnologías y recopilar información.
HIBP (Have I Been Pwned):
Servicio que permite a los usuarios verificar si sus datos personales han sido comprometidos en brechas de seguridad.
Typosquatting:
Una forma de ciberocupación que se basa en errores tipográficos cometidos por los usuarios de Internet.
CVE (Common Vulnerabilities and Exposures):
Un diccionario de vulnerabilidades de seguridad de la información conocidas públicamente.
Configuración de Seguridad:
Ajustes y políticas implementadas para proteger un sistema o aplicación de amenazas.
